# ZhKeyesIM

## 项目简介

本项目旨在开发一款即时通讯（IM）软件，将平时所学的各项基础知识加以综合应用。系统架构如下：

1. **整体框架**
   - **服务端**：基于 Linux 平台，采用 TCP 协议和 Reactor 模型，实现账号服务、消息服务、文件传输服务等核心功能；底层网络通信复用本人此前开发的 **“ZhKeyesServer”** 项目模块。
   - **客户端**：基于 Windows 平台，采用三层架构——表现层（Qt GUI）、数据处理层和网络层；为后续跨平台支持，客户端仅使用 Qt 进行界面渲染，不依赖其网络或线程模块。
2. **分阶段演进**
   - **阶段一**：将所有功能集成到单一服务程序中，一次性实现账号管理、即时消息、离线缓存、文件传输等必备特性；
   - **阶段二**：拆分为若干微服务（如认证服务、聊天服务、文件服务），各模块之间通过 RPC 或其他轻量协议互相调用，以增强系统的可扩展性与可维护性。

C++ 的学习路径相对宽泛，很多初学者（包括我在最初入门时）在掌握了基础语法后，不知道下一步该如何深入，或者学到一定程度却缺乏项目来检验和巩固所学。为了解决这一困境，我计划将本项目打造成新手踏入 C++ 开发的“第一座灯塔”——以它为实践载体，系统化地整理出所需的核心知识点和我的开发思路，并以文字的形式详细呈现。希望能为初入行的新人提供清晰的学习路线和实战指导，让大家在实践中不断成长。

注：这里只包括一部分C++的客户端开发、和服务端开发的知识，不涉及到其他方向例如音视频、量化、游戏开发。但是其技术栈可以作为刚入门或者想要入门C++开发者的通用技术栈，在校大学生推荐大三甚至大二暑假就开始学习打好基础，如果是已经开始工作的，也可以作为工作项目之外的补充。

注：本项目为了串联所有的知识点，一些地方不会使用最佳的工程实践，比如没有使用一些成熟的网络通信库，当然我可以这么做，但是我觉得作为一个学习项目来讲，最重要的不是完成代码，而是检验理论。在你完全理解这个项目中的知识之后，你可以尝试用你喜欢的方式对其进行重构。

最后祝愿大家都能有所收获！
